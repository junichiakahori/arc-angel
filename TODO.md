## 📜 開発基本ルール (General Policy)
- [x] **変更管理**: 修正時は最新開発版（現在は `latest/v1.2.html`）のみを編集し、`index.html` は安定版 (v1.1) として保護する。
- [x] **日時の同期**: 修正時は必ず `version_history.json` の対象バージョンとファイル内時刻を**ユーザーの現在時刻**に合わせる（一分でも未来は厳禁）。
- [x] **自動プッシュ**: 修正後は必ず Git のコミット・プッシュを実行する。
- [x] **ファイル命名**: リソースはすべて **スネークケース (`snake_case`)**。
- [x] **楽曲並び順**: 楽曲追加時は必ず `songs.json` および内部バックアップリストを**リリース日順（昇順）**に維持する。
- [x] **譜面パスの事前定義**: 新曲追加時は `songs.json` に将来用の譜面ファイル名（例：`assets/charts/id_hard.txt`）をあらかじめセットする。
- [x] **MP4追加時の自動同期**: 新規に `.mp4` ファイルを追加した際は、即座に `songs.json` に同名の空の譜面エントリを追加し、整合性を保つ。
